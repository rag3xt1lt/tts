Найденные секреты / скрытые эндпоинты (по состоянию на текущий момент)
Проект: Black Barmen (клон API)

1) Скрытый эндпоинт: GET /secret
Источник: запуск discover.py по референсу (https://bar.antihype.lol)

Поведение референса:
- GET /secret без авторизации:
  - HTTP 401
  - JSON:
    {
      "detail": {
        "status": "error",
        "error": "unauthorized"
      }
    }

- GET /secret с авторизацией (Authorization: Bearer <token>):
  - HTTP 200
  - JSON:
    {
      "status": "error",
      "error": "not_found"
    }

Что сделано в локальном клоне:
- Добавлен эндпоинт GET /secret с ответом HTTP 200 и JSON { "status":"error","error":"not_found" } при наличии токена.
- Приведён формат 401 Unauthorized к виду референса: detail = {status:"error", error:"unauthorized"}.

2) Ограничение референса по частоте запросов (rate limit)
Наблюдение:
- При частом переборе путей референс начинает отвечать HTTP 429:
  {
    "status": "error",
    "error": "rate_limit",
    "retry_after": N
  }

Что сделано в инструментах:
- discover.py обновлён: 429 не считается “хитом”, скрипт ждёт retry_after и замедляет перебор.

Примечание:
- На момент фиксации найден только /secret; остальные кандидаты из wordlist упирались в rate limit либо возвращали 404.
